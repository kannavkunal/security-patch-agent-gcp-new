## Data Flow Diagram

### 1. PATCH Mode (Manual Scan)

```
User → Web UI → API (/scan) → Pub/Sub → Worker → K8s Job → [8 Phases] → GitHub PR → BigQuery
                                                              ↓
                                                         Vertex AI (Gemini)
                                                              ↓
                                                         Secret Manager
                                                              ↓
                                                            GCS
```

### 2. REVIEW Mode (Webhook Scan)

```
GitHub PR → Webhook → API (/webhook/github) → HMAC Validation → Pub/Sub → Worker → K8s Job → [Scan PR] → Comment on PR → BigQuery
```

### 3. Context Memory Flow

```
Current Scan → BigQuery (past scans) → Context Builder → Gemini (with history) → Better Patches
```

## Security Layers

```
┌────────────────────────────────────────────────────────────────┐
│  Layer 1: Network                                              │
│  - Load Balancer (public IP)                                   │
│  - GKE firewall rules                                          │
│  - HMAC signature verification (webhooks)                      │
└────────────────────────────────────────────────────────────────┘
                          ↓
┌────────────────────────────────────────────────────────────────┐
│  Layer 2: Authentication                                       │
│  - API Key validation (HMAC-SHA256)                            │
│  - Workload Identity (no key files)                            │
│  - GitHub PAT (limited scope: repo only)                       │
└────────────────────────────────────────────────────────────────┘
                          ↓
┌────────────────────────────────────────────────────────────────┐
│  Layer 3: Authorization                                        │
│  - Repository whitelist (VULNERABLE_REPOS)                     │
│  - IAM roles (least privilege)                                 │
│  - No auto-merge (human approval required)                     │
└────────────────────────────────────────────────────────────────┘
                          ↓
┌────────────────────────────────────────────────────────────────┐
│  Layer 4: Isolation                                            │
│  - K8s Jobs (separate pods per scan)                           │
│  - Resource limits (CPU/memory)                                │
│  - TTL cleanup (5 minutes)                                     │
│  - Namespace isolation                                         │
└────────────────────────────────────────────────────────────────┘
                          ↓
┌────────────────────────────────────────────────────────────────┐
│  Layer 5: Audit                                                │
│  - BigQuery audit trail                                        │
│  - Cloud Logging (all requests)                                │
│  - GCS evidence storage                                        │
└────────────────────────────────────────────────────────────────┘
```
