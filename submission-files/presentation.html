The Solution

"What if vulnerabilities fixed themselves?"

  • Detects vulnerabilities using Semgrep
  • Understands context using Gemini 2.5 Pro
  • Generates secure code fixes
  • Creates pull requests automatically
  • Documents everything for auditors

Two Operating Modes

🔧 PATCH Mode

Trigger: Manual API call

Scope: Full repository

Output: PR with automated fixes

✅ Proactive remediation

✅ AI-generated patches

✅ Evidence in GCS

🔍 REVIEW Mode

Trigger: GitHub webhook

Scope: PR changes only

Output: Security comments

✅ Only NEW vulnerabilities

✅ Zero false positives

✅ Blocks risky merges

System Architecture

GitHub → FastAPI → Pub/Sub → K8s Job → 8-Phase Pipeline ↓ Gemini 2.5 Pro Pipeline: ┌────────────────────────────────────────────┐ │ Phase 1: Repository Analysis │ │ Phase 2: Vulnerability Detection (Semgrep)│ │ Phase 3: Context (BigQuery historical) │ │ Phase 4: Patch Generation (Gemini AI) │ │ Phase 5: Verification (future) │ │ Phase 6: GitHub Integration (PR/comments) │ │ Phase 7: Audit Logging (BigQuery) │ │ Phase 8: Evidence (GCS markdown files) │ └────────────────────────────────────────────┘ Output: GitHub PR + GCS Evidence + BigQuery Analytics

Key Innovations

1. LLM-Powered Remediation

First tool using Gemini 2.5 Pro for full-file security patches

2. Historical Context Memory

Queries BigQuery for past scans to prevent regression bugs

3. Differential Analysis

Compares PR vs base branch → only reports NEW vulnerabilities

4. Audit-Grade Evidence

Automated CVSS scoring + attack scenarios for compliance

5. Event-Driven Scale

Kubernetes + Pub/Sub handles 100+ concurrent scans
